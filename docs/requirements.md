# Kotoba Quest 要件定義書

最終更新: 2026-04-13

---

## 1. プロジェクト概要

### 目的

子供たちの**発想力・創造性を育む**ことを主眼に置いた、プログラミング学習ゲーム。
既存サービスとの差別化は以下の通り。

| サービス | 特徴 | Kotoba Quest との差別化 |
|----------|------|------------------------|
| paiza | スキル判定・競技寄り | 正解を覚えるのではなく、自分で考えて試行錯誤する体験を重視 |
| Scratch | ビジュアルプログラミング寄り | テキストベースの論理的思考力を自然に習得させる |

### ターゲットユーザー

- **小中学生全体**
- 段階的な難易度設計により、プログラミング未経験者から中級者まで対応

---

## 2. プラットフォーム

| 項目 | 内容 |
|------|------|
| 動作環境 | Webブラウザ（PC / タブレット） |
| 料金 | 無料プレイ |
| ホスティング | Vercel（無料枠） |

---

## 3. ゲームシステム

### 3-1. ジャンル

ドラゴンクエスト的な**ターン制バトルRPG**。

### 3-2. バトル方式

**事前プログラミング型**

- バトル開始前にプレイヤーがコードを全部書く
- 書いたコードを「実行」するとバトルが始まり、結果が再生される
- バトル中にリアルタイムで操作することはない

### 3-3. プレイヤーリソース

| リソース | 説明 |
|----------|------|
| HP | ヒットポイント。0になるとゲームオーバー |
| MP | マジックポイント。魔法・特殊行動に消費する |

---

## 4. プログラミング言語仕様

### 4-1. 言語コンセプト

- **日本語**でPythonのように書ける独自言語
- 漢字・ひらがなをキーワードとして使用し、英語が苦手な小学生でも直感的に書ける

### 4-2. パーサー

| 項目 | 内容 |
|------|------|
| ライブラリ | **Ohm.js** |
| 採用理由 | Peggyより文法定義が直感的で、保守・拡張がしやすい |

---

## 5. パートナーキャラクター

- プレイヤーが詰まった時に相談できる**AIパートナーキャラクター**
- プレイヤーの書いたコードを読み取り、文脈に合ったアドバイスを提示する
- **AIと連携**（Claude API 想定）

---

## 6. 技術スタック

| カテゴリ | 技術 | 備考 |
|----------|------|------|
| フレームワーク | Next.js | Vercel との親和性が高く、無料ホスティングとの相性が良い |
| ホスティング | Vercel | 無料枠で運用 |
| 言語パーサー | Ohm.js | 独自日本語言語の構文解析 |
| AI連携 | Claude API（想定） | パートナーキャラクターの応答生成 |
| 言語 | TypeScript | - |

> **注意:** 現プロトタイプは Vite + Peggy 構成だが、本実装は Next.js / Ohm.js 前提で進める。
