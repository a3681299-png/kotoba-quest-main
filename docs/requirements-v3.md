## 1. プロジェクト概要

### 1-1. キャッチコピー
> 「自分の言葉で魔法を作り、自分のやり方で世界を救う」

### 1-2. 目的

子供たちの**発想力・創造性を育む**プログラミング学習ゲーム。
「正解を覚える訓練」ではなく、**自分で考えて試行錯誤する体験**を重視する。

### 1-3. 差別化軸

| サービス | 特徴 | Kotoba Quest との差別化 |
|---|---|---|
| Scratch | ビジュアルプログラミング | テキストベースの論理的思考力を自然に習得させる |
| paiza | スキル判定・競技寄り | 正解を覚えるのではなく、発想したことが武器になる体験 |
| **Kotoba Quest** | 日本語テキスト×RPG | 自分の戦略を通した快感 / 失敗→新しい作戦のきっかけ |

### 1-4. ターゲットユーザー

- **小中学生全体**を主対象とする
- 段階的な難易度設計により、プログラミング未経験者から中級者まで対応

| 想定層 | 年齢 | 対応Stage |
|---|---|---|
| 小学校低学年 | 6〜8歳 | Stage 1（ひらがな・1〜2行） |
| 小学校高学年 | 9〜12歳 | Stage 2〜3（繰り返し・変数まで） |
| 中学生 | 12〜15歳 | Stage 4〜6（条件分岐・合体魔法・総合） |

## 3. ゲームシステム

### 3-1. ジャンル

ドラゴンクエスト的な**ラウンド進行型バトル RPG**。
ターン制ではなく、プレイヤーが書いたループプログラムの各繰り返しが 1 ラウンドとなる。

### 3-2. バトル方式：事前プログラミング型（ループ駆動）

- バトル開始前にプレイヤーがコードを全部書く
- 「実行」するとバトルが始まり、結果が再生される（バトル中のリアルタイム書き換えなし）
- プレイヤーは**メインループ**を自分で記述し、ループの各繰り返し（ラウンド）が 1 コマとなる
- ループ内のプレイヤー行動が終わると、敵が 1 回行動する
- 各ラウンド開始時に**変数リセット**と**MP回復**が行われる

```
繰り返す(敵が生きている あいだ):
  もし 自分のMP が 80 以上 ならば:
    魔法(フレイム)
    魔法(アクア)
    魔法(スパーク)
  そうでなければ:
    防御()
```

### 3-3. プレイヤーリソース

| リソース | 説明 | 初期値（Stage 1） | 最終値（Stage 6） |
|---|---|---|---|
| HP | 0 になるとゲームオーバー | 100 | 100（全ステージ共通） |
| 最大MP | 魔法に消費。毎ラウンド開始時に増加+回復 | 50 | 100 |
| 攻撃力 | 魔法ダメージの基準値 | 20 | 45 |

#### MP 詳細仕様

- 戦闘開始時の現在 MP = その Stage の初期最大 MP と同値
- **各ラウンド開始時**: 最大 MP を +10 増加 → その後、増加後の最大 MP の 1/3（切り捨て）を回復
- 例（Stage 1）: 1R = 最大50・現在50 / 2R = 最大60・+20回復 / 3R = 最大70・+23回復

#### 消費 MP 一覧

| 魔法種別 | 消費 MP |
|---|---|
| 単属性魔法（1属性） | 10 |
| 3属性合体魔法 | 80 |
| 4属性合体魔法 | 100 |
| 5属性合体魔法 | 120 |
| 防御() / 回復() / 待機() | 0 |

### 3-4. Wave 間のステータス引き継ぎルール

- **HP**: 次 Wave 開始時に **100 にリセット**（Wave クリアごとに全回復）
- **MP**: 次 Wave 開始時に **その Stage の初期最大 MP にリセット**
- **最大 MP**: 次 Wave 開始時に **その Stage の初期最大 MP にリセット**
- **変数**: ラウンドごとにリセット（次のラウンドへ持ち越さない）
- **同 Wave 内の複数敵戦闘**（順次戦闘）: HP/MP を引き継ぎ + ボーナス回復（+20HP）

### 3-5. 属性相性

**循環型**（5 属性）:
```
水 → 火 → 氷 → 風 → 雷 → 水
（矢印の向き: 左の属性が右の属性タイプの敵に強い）
```

**倍率**: 弱点 **2x** / 通常 **1x** / 耐性 **0.5x**

| 攻撃魔法 ＼ 敵タイプ | 火型 | 水型 | 雷型 | 氷型 | 風型 | 無属性 |
|---|---|---|---|---|---|---|
| フレイム（火） | 1x | 0.5x | 1x | **2x** | 1x | 1x |
| アクア（水） | **2x** | 1x | 0.5x | 1x | 1x | 1x |
| スパーク（雷） | 1x | **2x** | 1x | 1x | 0.5x | 1x |
| フロスト（氷） | 0.5x | 1x | 1x | 1x | **2x** | 1x |
| ゲイル（風） | 1x | 1x | **2x** | 0.5x | 1x | 1x |

- 吸収（ダメージが敵 HP を回復）は**敵固有の特殊設定**として個別定義
- 吸収 HP 回復量 = 本来発生するはずだったダメージ量と同値

#### 属性解放スケジュール

| 初期解放 | Stage 2 クリア報酬 | Stage 3 クリア報酬 |
|---|---|---|
| 火・水・雷 | 氷 解放 | 風 解放 |

### 3-6. 合体魔法

#### 発動条件

- 属性の複合は**3種類以上のみ**（2 属性合体は採用しない）
- 判定タイミング: **ラウンド終了時にまとめて判定**
- 同一ラウンド内で 3 種類以上の異なる属性の `魔法()` が実行され、かつ**必要 MP が足りている場合**に自動発動
- 同一属性を複数回使っても **1 種類としてカウント**
- MP 不足の場合は合体せず単属性として個別処理

#### ダメージ計算

```
合体魔法ダメージ = max(1, floor(攻撃力 × 属性数 × 最大属性倍率) - 敵防御力) + 特殊効果
```

- `属性数` = 合体に使用した異なる属性の種類数（3・4・5）
- `最大属性倍率` = 合体に使用した属性のうち敵に対して最も高い属性倍率
- **5 属性合体魔法**は耐性貫通として**固定 200 ダメージ**（Stage 4 Wave 3 で意図する一掃体験のため）

#### 特殊効果の種類

**状態異常（敵へ）**

| 効果名 | 内容 | 継続 |
|---|---|---|
| 燃焼 | 毎ラウンド開始時に敵 10 ダメージ | 3 ラウンド |
| 感電 | 敵の攻撃ダメージを半減 | 2 ラウンド |

**デバフ（敵へ）**

| 効果名 | 内容 | 継続 |
|---|---|---|
| 防御力低下 | 敵の防御力が 0 になる | 2 ラウンド |
| 攻撃力低下 | 敵の攻撃ダメージが 25% 減 | 3 ラウンド |

**バフ（プレイヤーへ）**

| 効果名 | 内容 | 継続 |
|---|---|---|
| 攻撃力アップ | プレイヤーの攻撃力 +10 | 2 ラウンド |
| HP回復 | 即座に 30HP 回復 | 即時 |
| 防御アップ | 被ダメージ固定軽減値 +15（防御()と重複可） | 2 ラウンド |
| 全体攻撃 | Wave 内の全敵に同時ヒット（各敵に独立計算） | 即時 |

#### 合体魔法 16 種一覧

**3属性合体（10 種）**

| 属性組み合わせ | 名前 | 特殊効果 |
|---|---|---|
| 火・水・雷 | 未定 | 燃焼 + 感電 |
| 火・水・氷 | 未定 | HP回復 + 防御力低下 |
| 火・水・風 | 未定 | 全体攻撃 |
| 火・雷・氷 | 未定 | 感電 + 攻撃力低下 |
| 火・雷・風 | 未定 | 燃焼 + 攻撃力アップ |
| 火・氷・風 | 未定 | 防御アップ + 燃焼 |
| 水・雷・氷 | 未定 | 感電 + HP回復 |
| 水・雷・風 | 未定 | 全体攻撃 + 感電 |
| 水・氷・風 | 未定 | HP回復 + 攻撃力低下 |
| 雷・氷・風 | 未定 | 感電 + 防御アップ |

**4属性合体（5 種）**

| 属性組み合わせ | 名前 | 特殊効果 |
|---|---|---|
| 火・水・雷・氷 | 未定 | 燃焼 + 感電 + 防御力低下 |
| 火・水・雷・風 | 未定 | 全体攻撃 + 感電 + 攻撃力アップ |
| 火・水・氷・風 | 未定 | 全体攻撃 + HP回復 + 防御アップ |

| 水・雷・氷・風 | 未定 | 全体攻撃 + HP回復 + 感電 |

**5属性合体（1 種）**

| 属性組み合わせ | 名前 | 特殊効果 |
|---|---|---|
| 火・水・雷・氷・風 | 未定 | **固定 200 ダメージ** + 全体攻撃 + 燃焼 + 感電 + 攻撃力アップ + HP回復 |

#### 通常版と省エネ版

- `魔法(魔法名)` と**直接魔法名を書く**のが通常版（フル威力・フルコスト）
- 魔法名を変数に入れ `魔法(変数名)` と書くのが**省エネ版**（MP 安め・威力低め）
- 省エネ版の数値はバランス調整フェーズで確定

#### バランス方針

- 基礎ステージでは**3 ラウンド後に合体魔法を撃てること**を基準
- 初期最大 MP 50 のとき**4 ラウンド目開始時に初めて 3 属性合体魔法を使用可能**な設計

### 3-7. プレイヤー基本コマンド

| 命令 | 記法 | 説明 |
|---|---|---|
| 魔法 | `魔法(魔法名)` | 指定した単属性魔法を使用 |
| 防御 | `防御()` | そのラウンドの被ダメージを -10 軽減 |
| 回復 | `回復()` | HP を最大 HP の 30%（30）回復。**1 ラウンド 1 回まで**（繰り返し内でも 1 回） |
| 待機 | `待機()` | 行動をスキップ |

- プレイヤーは**魔法のみ**で戦う（物理攻撃 `攻撃()` はコマンドセットに含まない）

### 3-8. 敵情報の開示（段階的露出）

敵情報は 3 段階で管理：
1. **最初から見える情報**（名前・HP・属性タイプ）
2. **見た目・演出・固定ヒントで段階的にわかる情報**（特殊行動・吸収条件など）
3. **内部に持つが直接は見せない情報**（攻撃パターン乱数・適応ロジック詳細）

- 分析コマンドは初期仕様では**採用しない**
- 一度理解した敵情報は**敵メモ・図鑑**として保持できることが望ましい（将来機能）

### 3-9. エラー時のヒント設計

#### コード入力エラーの種類

| エラー種別 | 検知タイミング | 表示方法 |
|---|---|---|
| 構文エラー | パース時 | エディタにエラー位置と原因を表示 |
| 実行時エラー | バトル開始時 | エラー内容と該当行をハイライト |
| 戦術エラー（敗北） | バトル終了時 | 「リトライ」ボタンと固定ヒント |

#### 段階的ヒント（敗北時）

| 失敗回数 | ヒントレベル | 内容の方針 |
|---|---|---|
| 1 回目 | 気づきを促す | 状況に応じた固定メッセージ（「MP が途中で尽きたみたい」等） |
| 2 回目 | 具体的な誘導 | コストや回数の計算を促す固定メッセージ |
| 3 回目 | ほぼ答え | 数値を直接示す固定メッセージ |

- ヒントはすべて**ローカル処理**で生成（外部 API 呼び出しなし）
- リトライ時は**コード入力からやり直し**（バトル途中復帰なし）

#### 学習型ラスボスの適応ロジック

集計範囲: **Stage 1〜5 の全クリアデータ**（localStorage に永続化）
適応タイミング: **Wave 5 開始時に一括計算**してボススタータスを確定（バトル中は固定）

```ts
interface ActionHistory {
  magicUsage: Record<MagicName, number>;  // 属性ごとの使用回数
  comboCount: number;                      // 合体魔法発動回数
  defendCount: number;                     // 防御()使用回数
  healCount: number;                       // 回復()使用回数
  totalRounds: number;                     // 総バトルラウンド数
  totalBattles: number;                    // 総バトル数
}
```

| 条件 | 適応内容 |
|---|---|
| 最多使用属性 上位 2 種 | その属性のダメージ 0.5x |
| `comboCount / totalRounds ≥ 0.15` | 合体魔法ダメージ × 0.7 |
| `(defendCount + healCount) / totalRounds ≥ 0.30` | チャージサイクル 3ラウンド → 2ラウンドに短縮 |

- 適応内容はセリフで**間接的に**伝える（「お前のフレイムは見切った」等）

## 5. 言語仕様（概要）

> 詳細仕様は `../prototype-5173/docs/language-spec.md` を正本とする。

### 5-1. コンセプト

- **日本語で Python のように書ける独自言語**
- 漢字・ひらがなをキーワードに使用し、英語が苦手な小学生でも直感的に書ける
- 上から下へ順に実行（基本）
- 学習順: `順次実行 → 繰り返し → 変数 → 条件分岐` に沿って段階的に理解できる設計

### 5-2. 構文アンロック

| Stage | 解放される構文 |
|---|---|
| 1 | `魔法(魔法名)` `防御()` `回復()` `待機()` `繰り返す(条件 あいだ):` |
| 2 | `繰り返す(N):`（回数指定ループ）|
| 3 | 変数代入 / `プレイヤー.変数名` / `なかま.変数名` 参照 |
| 4 | `もし 条件 ならば:` / `そうでなければ もし:` / `そうでなければ:` |
| 5〜6 | 全構文自由 |

### 5-3. 基本構文（要約）

**プログラム構造**

```
繰り返す(敵が生きている あいだ):   # メインバトルループ
  繰り返す(3):                      # ラウンド内の回数指定ループ
    魔法(フレイム)
  もし 自分のMP が 80 以上 ならば:
    魔法(アクア)
  そうでなければ:
    防御()
```

**比較演算子**

| 記法 | 意味 |
|---|---|
| `より小さい` | `<` |
| `より大きい` | `>` |
| `以上` | `>=` |
| `以下` | `<=` |
| `と等しい` | `==` |
| `と等しくない` | `!=` |

**論理演算子**

| 記法 | 意味 |
|---|---|
| `かつ` | `and` |
| `または` | `or` |
| `ではない` | `not` |

**条件に使えるゲーム状態参照**

| 参照 | 意味 |
|---|---|
| `敵が生きている` | 敵の HP が 0 より大きい |
| `敵のHP` | 対象敵の現在 HP |
| `自分のHP` | プレイヤーの現在 HP |
| `自分のMP` | プレイヤーの現在 MP |
| `敵が火状態`〜`敵が風状態` | 敵の現在状態（Stage 4 以降） |

**ターゲット指定**

```
敵[1番目]へ 魔法(フレイム)
敵[2番目]へ 魔法(アクア)
魔法(スパーク)   # 指定なし → 先頭の生きている敵を自動ターゲット
```

**NPC との相互参照**（Stage 3 以降）

```
# NPC コード（マリア）
まほう = フレイム

# プレイヤーコード
もし なかま.まほう と等しい フレイム ならば:
  魔法(アクア)
```

- 参照は**読み取り専用**（代入不可）
- 参照タイミング: **ターン開始時点の値**（循環依存防止）
- 変数のみ参照可（HP / MP / 状態の相互参照は不可）

### 5-4. 実装前提

- **ブロック構造**: インデント型（Python 式）。スペース 2 個を基本
- **ブロック開始マーカー**: `:` をブロック先頭行の末尾に付ける
- **全角スペース対応**済み（子供が全角で入力しても通る）
- **最大ループ回数**: 50 ラウンドで強制終了（無限ループ防止）
- **コメント**: 初期実装では扱わない
- パーサーの正本: **Ohm.js**
- エディタ: **CodeMirror 6**（正誤判定の正本は Ohm.js）
- 将来的に構文エラーと実行時エラーを区別できることが望ましい

## 7. 非機能要件

## 8. セーブデータ・履歴仕様

### 8-1. 保存対象

| データ | 保存先 | 用途 |
|---|---|---|
| クリア済みステージ | localStorage | ステージ選択画面の解放制御 |
| 解放済み属性 | localStorage | 使用可能魔法リストの表示制御 |
| 行動履歴（ActionHistory） | localStorage | Stage 6 ラスボスの適応計算 |
| 現在編集中のコード | Zustand state（揮発） | バトル画面内のドラフト保持 |

### 8-2. リセット仕様

- **ステージリセット**: 各ステージのプレイは何度でもやり直し可能（クリア状態は維持）
- **進捗リセット**: タイトル画面の設定から「すべての進捗をリセット」が可能（要二重確認）
- **行動履歴のみリセット**: 設定から「ラスボスの適応をリセット」が可能（履歴のみクリア）

### 8-3. 端末をまたいだデータ移行

- MVP フェーズでは**サポートしない**（同一ブラウザでのみ有効）
- 将来的にアカウント機能と連携した移行を検討

#### ステージ結果画面

- 属性解放演出（該当時）
